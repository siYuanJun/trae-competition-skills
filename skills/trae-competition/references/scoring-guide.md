# 大赛评分标准解读与高分策略

> 本文档分析 TRAE AI创造力大赛核心评分维度，并为每个维度提供可落地的优化策略。
> 配合 `scripts/scoring-optimizer.md` 使用，可对产出方案进行逐维评分和定向优化。

---

## 一、评分维度总览

| 维度 | 权重 | 核心考察点 | 当前 Skill 覆盖度 |
|------|------|-----------|------------------|
| 创新性 | 35% | 需求洞察、解决思路独创性、技术运用新颖度 | ⚠️ 薄弱（未设创新约束） |
| 实用性 | 30% | 场景真实度、解决效果、落地可行性 | ⚠️ 一般（模板填痛点，缺少深度） |
| 完成度 | 20% | 核心功能链路（3-5步闭环）、用户体验流畅 | 🟡 一般（仅做格式检查） |
| 表现力 | 15% | 视觉设计质量、记忆点交互、信息呈现效果 | 🔴 薄弱（设计规范过于笼统） |

> **核心认知**：这不是软件工程大赛，是「创意原型+AI提效能力」比拼。纯开发/技术部分只占15%，创意和体验才是胜负手。花10天写后端，不如花3天把核心流程做流畅、把文案写到位、把视觉调舒服。

---

## 二、各维度高分策略

### 2.1 创新性（35%）：如何让方案脱颖而出（核心胜负手）

**评分倾向**：评委寻找「没想到还能这样」的惊喜感，而非「换了个壳的常见工具」。

**高频低分陷阱**：
- 换皮型：把 Excel 做成网页版计算器，换个配色
- 资讯型：把公众号文章整合成静态列表页，无交互
- 大而空：AI 解决一切养老问题 → 一句口号，没有核心功能
- 窄切口不足：想做「全场景平台」，每个功能都只做了个壳

**高分策略**：

| 策略 | 说明 | 示例 |
|------|------|------|
| 交叉创新 | 两个领域交叉产生新视角 | 反诈 + 游戏化闯关 = 沉浸式反诈模拟器 |
| 场景降维 | 宏大问题中切出极细分、一个HTML能真正解决的场景 | 不是「帮助老年人」而是「帮老人识别诈骗短信中的链接」 |
| 交互创新 | 改变信息呈现和交互方式 | 地铁图式社区资源导航、树状问答式政策解读 |
| 数据挖掘 | 让静态数据动态化、可视化 | 把 CPI 数据做成可拖拽时间线对比图 |

**选择优先级**：交叉创新 > 场景降维 > 交互创新 >= 数据挖掘 > 单领域简单迭代 > 换皮型方案

**提示词中的落地**：在 `select-topics.md` 中增加创新性约束和窄切口约束，要求每个选题提供「为什么这个角度别人没想到」的创新点说明，以及3-5步MVP路径。

---

### 2.2 实用性（30%）：让评委觉得「这个我也想用」

**评分倾向**：真问题 + 真场景 + 真能用。评委代入用户视角判断「如果我是目标用户，会不会用」。

**高频低分陷阱**：
- 空气痛点：宣称解决的问题实际不存在（如「大学生找不到课程表」）
- 无场景感：只说帮助了谁，说不清在什么情况下、怎么用的
- 不可落地：功能描述宏大，但一个 HTML 页面根本承载不了

**高分策略**：

| 策略 | 说明 | 示例 |
|------|------|------|
| 故事驱动 | 用一个具体人物的一天串联使用流程 | 「张阿姨早上打开页面→输入症状→获得附近医院推荐→一键导航」 |
| 对比论证 | 有/无本方案前后对比 | 「以前需要下载3个App、切换5次，现在一个页面搞定」 |
| 数据佐证 | 引用公开数据支撑痛点真实性 | 「据2025年公安部数据，养老诈骗年均涉案金额超X亿」 |
| MVP思维 | 只做最核心功能，但做透做深 | 不做大而全的养老平台，只做「养老金计算+领取指南」 |

**提示词中的落地**：在 `generate-post.md` 中要求文案包含「用户使用故事」和「有无对比」段落。

---

### 2.3 完成度（20%）：让提交物经得起推敲

**评分倾向**：核心流程能跑通（3-5步闭环）、没有明显断点、用户能无障碍完成主任务。宁可只做3步完整流程，不做10步半成品。

**高频低分陷阱**：
- 功能半成品：有按钮但点了没反应，有输入框但不能提交
- 逻辑断裂：从 A 步骤直接跳到 C 步骤，中间缺失过渡
- 价值模糊：做完看完不知道这网页到底帮我解决了什么

**高分策略**：

| 策略 | 说明 |
|------|------|
| 闭环设计 | 从用户进入页面到达成目标，每一步都可以完成 |
| 反馈机制 | 每个操作有视觉反馈（按钮变色、动画提示、结果展示） |
| 核心链路优先 | 宁可只做3步完整流程，不做10步半成品 |
| 自检清单 | 按核心链路逐步骤点击，确认无卡顿 |

**提示词中的落地**：在 `quality-check.md` 中增加完成度检查项；在 `scoring-optimizer.md` 中模拟用户走查。

---

### 2.4 表现力（15%）：让作品看着像个正经产品

**评分倾向**：不是比炫技，而是比「视觉是否服务于功能」。干净、克制、有设计感 > 花哨堆砌。

> **核心策略**：页面完成感比技术实现更重要。即使背后全是静态数据，也要让页面看起来是完整可用的产品。

**高频低分陷阱**：
- 默认样式裸奔：浏览器默认字体、默认蓝色链接、无间距排版
- 配色灾难：红绿配、对比度不足、多彩混乱
- 交互割裂：PC 看着还行，手机端布局全崩

**高分策略**：

| 策略 | 说明 |
|------|------|
| 色彩克制 | 主色 + 辅色不超过 3 种，灰度系统布局 |
| 间距呼吸 | 卡段之间有足够留白，信息不拥挤 |
| 层级清晰 | 字号、粗细、颜色三层区分信息优先级 |
| 动效点睛 | 微小过渡动画（0.2-0.3s），不喧宾夺主 |
| 移动端优先 | 先按 375px 宽度设计，再适配 PC |
| **记忆点交互** | 每个页面至少有一个让人印象深刻的交互亮点（翻转/弹射/滚动效果） |

**提示词中的落地**：在 `generate-html.md` 和 `references/design-guide.md` 中指定详细设计规范，并新增记忆点交互要求和数据模拟策略。

# 三、绝对不用做的5件事（评审不看，纯浪费时间）

以下内容哪怕完全不做，也不影响获奖。很多选手踩坑就是把精力花在了这里：

1. **用户账号系统、注册登录、权限管理**
   - 纯工具原型完全不需要登录，评委打开就能体验核心功能反而是加分
2. **后端架构、数据库设计、并发性能优化**
   - 赛事不看工程化能力，所有数据用静态模拟即可
3. **全功能覆盖、多场景闭环**
   - 不要做大而全平台，把1个场景做透比10个场景做一半强10倍
4. **多端适配、移动端兼容、响应式布局**
   - PC端能正常打开体验就足够
5. **真实商业落地、盈利模式设计**
   - 创意赛不要求能赚钱，用户价值讲清楚即可

## 四、评分优化建议的优先级

1. **P0（立即见效）**：升级 `generate-html.md` 设计规范 → 直接提升表现力 + 完成度
2. **P1（高分核心）**：升级 `select-topics.md` 创新性约束 → 避免「选题平庸」这个最大失分点
3. **P2（锦上添花）**：新增 `scoring-optimizer.md` 评分预检 → 模拟评委视角查漏补缺
4. **P3（系统提升）**：升级 `generate-post.md` 场景深度 → 让实用性论证更有说服力

## 五、短周期出高分核心思路

### 重"体验包装"，轻"底层实现"
- 能用静态数据模拟的，就不接真实接口
- 能单页完成的，就不做多页面跳转
- 能写死的逻辑，就不做动态计算

### 窄切口，做深不做广
- 错误思路：AI解决一切问题（大而空）
- 正确思路：就做一个极细分场景的3-5步完整体验

### AI工具用在刀刃上
- 页面生成、文案撰写、图标设计、样例数据生成，全部交给AI完成
- 你只负责定方向、调细节、串流程
