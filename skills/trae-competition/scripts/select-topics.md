# 任务：筛选参赛创意选题（支持全赛道）

适用赛道：【学习工作、生活娱乐、社会服务、社会公益、硬件交互】

## 赛道方向说明
1. 学习工作：效率工具、知识管理、协作办公、技能学习等
2. 生活娱乐：休闲娱乐、生活服务、兴趣爱好、社交互动等
3. 社会服务：政务服务、便民服务、社区服务、公共服务等
4. 社会公益：公益科普、弱势群体关怀、环保公益、社会救助等
5. 硬件交互：IoT设备控制、智能硬件联动、传感器应用等

## 约束条件

### 合规约束（必须满足）
1. 贴合2026国内民生政策与市场需求：适老化、全民反诈、乡村振兴、无障碍、低碳环保、青少年心理健康科普、数字经济等；
2. 产物为单页HTML交互网页，TRAE Work Auto模式一键生成，无复杂后端逻辑；
3. 正向积极题材，无商业营销、无敏感内容，审核零驳回风险；
4. 创意痛点清晰，容易填充完整4模块提案，正文轻松超过100字；

### 创新性约束（高分必需）
5. **拒绝换皮型方案**：不是「把Excel/问卷做成网页版」「把文章列表做成静态页」这类常见套路；
6. **必须有差异化视角**：每个选题需说明「为什么别人没想到这个角度」或「这个切入点和常见方案有什么不同」；
7. **交叉创新优先**：两个领域交叉产生新视角（如反诈+游戏化、适老化+语音交互）优先于单领域迭代；
8. **场景降维优先**：从一个宏大问题中切出极细分的、可被一个HTML页面真正解决的具体场景；

## 输入参数
{指定目标赛道，如：社会服务、学习工作等，不指定则默认覆盖全部方向}

## 输出要求
输出4个独立选题，每个选题包含：
1. **创意名称**：简洁有力，一听就懂
2. **核心解决痛点**：具体的、可验证的真问题（非空气痛点）
3. **创新性亮点**：为什么这个角度独特？和常见方案有何不同？
4. **HTML页面核心功能**：MVP级别，只做核心但做透
5. **参赛加分优势**：贴合哪个政策方向/社会热点
6. **创新性评级**：⭐（中规中矩）到 ⭐⭐⭐⭐⭐（惊艳独创）
