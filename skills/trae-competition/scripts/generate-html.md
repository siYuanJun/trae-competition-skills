# TRAE Work Auto模式 HTML生成提示词

## 核心定位声明
> **这不是一个完整应用，而是高完成度的可交互原型/产品演示片。** 
> 核心原则：让评委看起来很厉害、用起来很顺畅，至于背后是否有复杂逻辑，不重要。
> 页面完成感比技术实现更重要——即使全是静态数据，也要看起来是完整可用的产品。

## MVP路径约束
- 核心用户路径不超过5步，3步最佳
- 每个步骤必须有明确的交互反馈（点击 → 变化 → 结果）
- 不做多页面跳转，单页完成全部流程

## 数据与内容策略
- **所有数据使用静态模拟**：不接真实API、不做真实后端、不用数据库
- 样例数据要看起来真实可信（使用合理的人名、数值、场景描述）
- 严禁出现「暂无数据」「示例数据仅供展示」等提醒文字——要让页面看起来真的能用

## 页面定位
根据选题类型定位为轻量化科普/展示/工具单页，适配手机+电脑双端响应式

## 设计规范（必须遵守）
参考 `references/design-guide.md` 中的详细规范，核心要求：

### 色彩
- 主色+辅色不超过3种，灰度系统布局
- 按赛道应用差异化配色（社会服务/公益=暖色、学习工作=冷色、生活娱乐=活泼、硬件=科技暗色）

### 字体
- 标题 24-32px / 副标题 18-20px / 正文 15-16px（适老类≥18px）
- 行高 ≥1.6，确保阅读舒适
- 字体栈：`-apple-system, "PingFang SC", "Microsoft YaHei", sans-serif`

### 间距
- 页面内边距 16-20px（手机）/ 24-32px（PC）
- 卡片间距 16-20px，段落间距 16-24px
- 按钮内边距 ≥10px 垂直 / 24px 水平

### 交互
- 按钮点击有 scale(0.97) 反馈
- 输入框聚焦有边框变色过渡（0.2s ease）
- 结果展示有淡入动画（opacity 0→1, 0.3s ease）
- 移动端触控区域 ≥44×44px

### 记忆点交互（每个页面至少实现1个）
- 翻转卡片展示详情（card flip）
- 滚动触发动画（scroll-triggered animation）
- 弹射/跳动结果展示（pop/bounce on complete）
- 一键主题切换（light/dark mode toggle）
- 进度可视化（progress bar/steps indicator）

### 无障碍
- 所有图片含 alt 属性
- 颜色不是唯一信息区分方式
- 适老场景对比度 ≥4.5:1

### 代码质量
- 单文件 HTML+内置 CSS/JS，无外部依赖
- CSS 变量管理颜色/间距/圆角
- 中文注释清晰，按区块组织
- 移动端优先，≥768px / ≥1024px 断点适配

### 禁止事项
- 外部 CDN 依赖（通用图标库除外，须注明）
- 内联样式散落（统一放 `<style>` 块）
- 花哨动画（单个动效 ≤0.3s）
- **真实API调用、后端接口、数据库操作**
- **用户注册、登录、权限验证**
