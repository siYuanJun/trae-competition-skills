# 赛事发帖文案生成提示词

前置规则：大赛官方发帖模板、审核规则、支持全赛道【学习工作、生活娱乐、社会服务、社会公益、硬件交互】

输入素材：{替换为单个创意选题信息，含创新性亮点和MVP路径}

## 核心文案定位
这是「创意原型+AI提效能力」大赛，不是软件工程大赛。文案的核心任务是讲清楚「给谁用、解决什么痛点、核心功能是什么、带来什么好处」——让评委看完能想象出产品画面，而非理解技术原理。

## 文案语言风格规范
- **轻量化表述**：强调「几步搞定」「一个页面就够」「打开即用」等原型友好措辞
- **体验导向**：多写「用户感受」「使用场景」「前后对比」，少写「技术实现」「架构设计」
- **口语化但不随意**：让文案读起来像在讲故事，不是技术说明书

## 强制输出内容

### 1. 合规标题
【对应赛道名称】+ 创意名称

### 2. 标签勾选说明
基础赛道选对应赛道，附加社会公益标签（如适用）

### 3. 完整正文（≥120字，严格4大模块，每个模块内部细分小点全部写满）

**模块1：创意名称 + 介绍**
- 1-2句话描述要解决的具体痛点（数据佐证更佳）
- 说明创意灵感、创作动机，以及**为什么这个角度别人没想到**（创新性）
- **产品形态定位**：明确指出「这是一个可交互HTML原型，不是商业级完整应用，但核心流程可走通、体验完整」
- 一句话概括产品形态（轻量化网页交互工具/可视化科普页，适配TRAE Work导出HTML，三步搞定）

**模块2：目标用户 + 痛点**
- 精准描述核心受益用户群体（年龄、身份、特征）
- **用户使用故事**：用一个具体人物的一天串联使用流程（如「张阿姨早上打开页面→看到3个诈骗案例→选择最像的一类→获取防骗口诀→转发给老姐妹」），故事必须包含「具体人物 + 具体场景 + 前后变化」
- **落地触达路径**（新增必填）：说明用户如何找到并使用这个产品，列出至少3个触达渠道（如：子女转发微信、社区公众号嵌入、公告栏二维码、志愿者引导等）
- **有无对比**：明确写出无本产品时用户的困境 vs 有本产品后的改善（如「以前收到诈骗短信只能凭感觉判断，现在打开页面3步就能识别」）

**模块3：价值与意义**
- 社会价值角度：受益人群规模、贴合政策方向
- 实用价值角度：效率提升、成本降低、体验改善
- 如有公开数据可引用作为论据支撑
- **体验价值**：强调「一个页面就能搞定」「打开即用无需下载」「3步完成核心操作」等轻量化优势

**模块4：TRAE Work HTML 物料说明**
- Auto模式生成流程简述
- 页面核心交互功能（按赛道指定设计要求）
- **页面亮点**：突出记忆点交互（如「翻转卡片展示」「滚动触发动画」「一键切换深色模式」）
- 跨设备适配说明
