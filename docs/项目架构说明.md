# 项目架构说明

> 面向想了解内部结构的开发者。

## 设计理念

本项目遵循「Skill 工程」的事实标准：一个 Skill = 一个目录，目录内含 `SKILL.md` 作为入口，`scripts/` 存放可调用的分步工具，`references/` 存放参考数据。

这样设计的好处：
- **AI 自动发现**：AI 助手会扫描 `skills/*/SKILL.md` 识别所有可用 Skill
- **按需加载**：AI 只在需要时读取 `scripts/` 下的子文件，避免一次加载过多内容
- **职责分离**：5个提示词独立维护，改文案不影响选题

## 目录结构

```
trae-competition-skills/
├── .github/workflows/validate.yml  # CI 自动校验
├── skills/trae-competition/        # ★ Skill 主体
│   ├── SKILL.md                    # 总调度指令（AI 入口）
│   ├── scripts/                    # 分步工具提示词
│   │   ├── select-topics.md        # 选题筛选
│   │   ├── generate-post.md        # 文案生成
│   │   ├── generate-html.md        # HTML 物料
│   │   └── quality-check.md        # 合规质检
│   └── references/                 # 参考资料
│       ├── competition-rules.md    # 赛事规则摘要
│       └── track-guide.md          # 赛道详细说明
├── docs/                           # 中文文档
│   ├── 快速上手指南.md
│   ├── 项目架构说明.md（本文档）
│   └── 提示词方法推广文章.md
├── scripts/                        # 工具脚本
│   ├── validate-skills.sh          # 结构校验
│   └── pack-release.sh             # 发布打包
├── README.md                       # 仓库首页
├── CONTEXT.md                      # AI 加载上下文
├── CHANGELOG.md                    # 版本记录
├── LICENSE                         # MIT 协议
├── package.json                    # 项目元信息
└── .gitignore
```

## 数据流转

```
用户输入：「用标准模式，赛道选社会服务」
        │
        ▼
   SKILL.md（总调度）
        │
        ├─ 阶段1 ──▶ select-topics.md ──▶ 4个选题
        │
        ├─ 阶段2 ──▶ generate-post.md ──▶ 报名帖文案
        │
        ├─ 阶段3 ──▶ quality-check.md ──▶ 合规定稿
        │
        ├─ 阶段4 ──▶ generate-html.md ──▶ HTML指令
        │
        └─ 阶段5 ──▶ 汇总交付清单
```

## 如何新增一个工具

在 `skills/trae-competition/scripts/` 下创建新 `.md` 文件，格式如下：

```markdown
---
name: tool-name
description: 工具的中文描述
---

# 工具的标题

工具的提示词正文...
```

然后更新 `SKILL.md` 中的工具表，添加新条目即可。

## 如何适配新的 AI 平台

SKILL.md 使用标准 Markdown 格式，兼容主流 AI 工具：

| 平台 | 加载方式 |
|------|----------|
| Trae | Skill 面板 → 导入目录 |
| Claude Code | `claude --load SKILL.md` |
| Cursor | 粘贴为 Rules / System Prompt |
| ChatGPT | 粘贴到 Custom Instructions |
| 通用 | 直接粘贴为对话上下文 |

## 校验机制

运行本地校验：
```bash
npm run validate
# 或
bash scripts/validate-skills.sh
```

校验内容：
- 所有必要文件是否存在
- SKILL.md 是否包含元数据头
- scripts/ 下的子文件格式是否正确

CI 会在每次 push 和 PR 时自动运行校验。
